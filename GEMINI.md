## 2. 開発の基本指針

*   **コンテキストの効率的な利用**: インポートされた規約を前提とし、冗長な説明を避けて実装に集中すること。
*   **副作用の最小化**: ドメインロジックは純粋関数として実装し、不変条件（Invariants）を常に担保すること。
*   **テストファースト**: 振る舞いの変更や修正を行う際は、インポートされたテスト規約に基づいたテストを必ず追加または更新すること。
*   **バージョンの維持**: ライブラリやツールのバージョンを独断で下げないこと。ダウングレードが必要な場合は、必ずユーザーの許可を得ること。
*   **コードフォーマットの強制**: すべての変更作業およびタスクの完了直前に、プロジェクトルートディレクトリで必ず `mise run format` を実行して、コードスタイルを統一すること。
*   **静的解析エラーの解消**: `mise run format` (detekt) の実行結果を必ず確認すること。`--auto-correct` で修正されないエラー（行数制限、複雑度、ネーミング規則など）が検出された場合は、手動で修正を行い、すべての警告・エラーを解消した状態でタスクを完了させること。
